# Code of Conduct

The maintainers of this project are committed to providing a friendly, safe,
and welcoming environment for all contributors, regardless of experience,
background, identity, or technical preferences.

## Expectations

- Be respectful in code review, issues, and chat. Critique the code or the
  idea, never the person.
- Assume good intent. If a comment reads harshly, ask before reacting.
- Treat operational mistakes as a chance to improve the system, not to assign
  blame. If a change you reviewed went wrong, own it together.

## Unacceptable behavior

- Personal attacks, harassment, or discriminatory language of any kind.
- Posting secrets (API tokens, private keys, customer data) in issues, PRs,
  or commit messages. If you need to share one for debugging, do it through
  a secure side channel.
- Knowingly bypassing review or CI to ship a change.

## Enforcement

Reports of unacceptable behavior can be sent privately to the maintainers
listed in [MAINTAINERS](MAINTAINERS). The maintainers will respond within a
reasonable timeframe and take action proportionate to the issue, up to and
including removal of contribution access.
